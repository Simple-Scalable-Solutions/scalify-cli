## Project Overview

`scalify-cli` is a Go CLI tool and MCP server for the **Scalify Platform** — generated by [cli-printing-press](https://github.com/mvanhorn/cli-printing-press) v4.2.2.

**Critical branding rule:** The CLI must never mention GoHighLevel, HighLevel, or LeadConnector anywhere in user-facing output, help text, docs, error messages, or flags. The underlying API provider is an internal implementation detail. All user-facing surfaces use "Scalify" exclusively.

## Architecture

Generated Go CLI project following printing-press conventions:

- `cmd/scalify-pp-cli/` — CLI entry point (Cobra)
- `cmd/scalify-pp-mcp/` — MCP server entry point
- `internal/cli/` — one file per resource + compound commands (`sync`, `search`, `workflow`, etc.)
- `internal/client/` — HTTP client wrapping the Scalify API; `authHeaderPrefix = "Bearer "` constant is required for dogfood auth-protocol check
- `internal/config/` — TOML config + env-var auth (`SCALIFY_TOKEN`); `AuthScheme = "Bearer"` constant
- `internal/mcp/` — runtime Cobra-tree MCP surface mirror
- `internal/store/` — SQLite cache with FTS5 full-text search

Config file: `~/.config/scalify-pp-cli/config.toml`  
SQLite DB: `~/.local/share/scalify-pp-cli/data.db`

## Auth

Set `SCALIFY_TOKEN=<api-token>` or run `scalify-pp-cli auth set-token`. The token is sent as `Authorization: Bearer <token>` on every request.

## Key Printing Press Conventions

- Auto-JSON output when piped (no `--json` flag needed)
- `--compact` returns only high-gravity fields (id, name, status, timestamps)
- `--agent` sets all agent-friendly defaults (`--json --compact --no-input --no-color --yes`)
- Typed exit codes: `0`=success, `2`=usage, `3`=not found, `4`=auth, `5`=API error, `7`=rate limited
- `--data-source auto|live|local` controls API vs cached reads

## Branding Invariant

The `internal/config/config.go` `BaseURL` points at the underlying API endpoint — this is intentional and must stay. It is never exposed in user-facing output. The `doctor` command intentionally omits the `base_url` field for this reason.

## Releasing

**Always use `--parallelism 2` when running goreleaser.** The config builds 2 binaries × 6 platforms = 12 concurrent Go compiler processes by default, which exhausts RAM and freezes the machine.

```shell
# Correct — always use this
goreleaser release --clean --parallelism 2

# Or via Makefile
make release
```

Never run `goreleaser release --clean` without `--parallelism 2`.
